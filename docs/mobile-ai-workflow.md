# スマホで永愛塾を回す手順

スマホでは「コードを書く」より、`AIに依頼する` と `見た目を確認する` を中心に回すのが相性いいです。

## 1. スマホでローカル画面を見る

PCで開発サーバーを立てます。

```bash
npm run dev:mobile
```

次に PC のローカル IP を確認します。

```powershell
ipconfig
```

`IPv4 アドレス` が `192.168.x.x` のように出るので、同じ Wi-Fi のスマホで次を開きます。

```text
http://192.168.x.x:3000
```

これでスマホから永愛塾の画面確認ができます。

## 2. スマホでは AI への依頼に寄せる

おすすめの使い方は次の流れです。

1. スマホで画面を見る
2. 気になった点を GitHub Issue に書く
3. AI に Issue の内容をそのまま渡す
4. 変更後に Preview またはローカル画面をスマホで再確認する

## 3. 依頼文の型

短くても、次の 4 つが入っていると AI がかなり動きやすいです。

1. どの画面か
2. 何が気になるか
3. どう変えたいか
4. 触ってほしくない範囲

例:

```text
対象: 設定画面
問題: 情報が多くて見づらい
希望: アカウント / プラン / 学習設定 / 連携 に分けたい
制約: インラインCSSのみ。課金処理は壊さない
```

## 4. GitHub Issue を使う

このリポジトリには、スマホからでも書きやすい `AI Task` の Issue テンプレートを追加しています。

おすすめの使い方:

1. GitHub で Issue を作る
2. `AI Task` を選ぶ
3. 画面名、問題、希望を埋める
4. その Issue を AI に渡す

## 5. スマホ運用で向いている作業

- 文言修正
- レイアウト整理
- ボタン導線の変更
- 画面単位の UI 修正
- バグ再現確認

## 6. PC でやった方がいい作業

- migration 適用
- Stripe webhook 確認
- 大きいリファクタ
- 型エラーや lint の最終確認

## 7. Stripe まわりの注意

スマホから画面確認はしやすいですが、Stripe の購入確認は PC 側で次を動かしておくのが安全です。

```bash
npm run stripe:listen
```

## 8. まず最初にやるとよいこと

1. `npm run dev:mobile`
2. スマホで `http://PCのIPv4:3000` を開く
3. 気になった画面を GitHub Issue に書く
4. AI に修正を依頼する

この流れが、いちばん無理なく回せます。
